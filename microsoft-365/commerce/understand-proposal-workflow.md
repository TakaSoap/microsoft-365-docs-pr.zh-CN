# <a name="understand-the-proposal-workflow"></a>了解建议工作流

建议是 Microsoft 为你的组织购买 Microsoft 产品和服务而提供的正式产品。 你直接与 Microsoft 代表合作，确定建议的特定产品、服务和条款。

Microsoft 代表起草一份提案，其中包含你和你的代表讨论的项目。 代表向你发送一封电子邮件，其中包含指向提案网站的链接。 该网站包含专门为你和你的组织准备的建议。

收到通知电子邮件后，请按照指向建议网站的链接进行操作。 登录到网站后，可以启动建议评审过程。

## <a name="prerequisites-for-buying-items-with-a-proposal"></a>使用建议购买商品的先决条件

在为建议购买项目之前，必须具有计费帐户和与 Microsoft 的协议。

### <a name="billing-account"></a>计费帐户

可以使用计费帐户来管理帐户设置、发票、计费配置文件以及产品和服务。 如果还没有计费帐户，Microsoft 代表会为你创建一个。 否则，只要你有权使用该计费帐户，他们就会为组织使用现有的计费帐户。

计费帐户权限由计费帐户所有者管理。 全局管理员可以将自己分配给计费帐户所有者角色，然后让其他人为帐户所有者计费。

有关计费帐户的详细信息，请参阅 [“管理计费帐户](manage-billing-accounts.md)”。

### <a name="microsoft-customer-agreement"></a>Microsoft 客户协议

MICROSOFT 客户协议 (MCA) 允许组织购买 Microsoft 产品和服务。 有关详细信息，请参阅[Microsoft 客户协议](https://www.microsoft.com/Licensing/how-to-buy/microsoft-customer-agreement)。

## <a name="permissions-needed-to-sign-an-agreement-or-pay-for-items"></a>签署协议或支付项目费用所需的权限

如果在计费帐户中没有分配的角色，则在查看建议时，会分配基本读取者角色。 此角色允许你查看但不采取任何操作来查看该建议。 必须先分配计费帐户所有者或计费帐户参与者角色，然后才能签署协议或购买产品和服务。 计费帐户所有者可以将此角色分配给你。

有关计费帐户角色的详细信息，请 [参阅了解对计费帐户的访问权限](manage-billing-accounts.md#understand-access-to-billing-accounts)。

如果这是一个新的计费帐户，并且没有人接受协议，则你将自动成为计费帐户所有者，前提是：

- 是建议中点名的人员， **还是**
- 已是组织的[Azure Active Directory全局管理员](/azure/active-directory/roles/permissions-reference#global-administrator)

## <a name="what-is-the-overall-workflow"></a>总体工作流是什么？

总体建议工作流如下所示：

- 你的 Microsoft 代表创建一个建议并在电子邮件中向你发送链接。
- 可以使用该链接转到建议登录页。
- 你将查看组织的信息。
- 查看建议，根据需要接受 MCA，然后完成结帐过程。
  > [!IMPORTANT]
  > 您必须有权代表你的组织签署 MCA。 如果没有该权限，则必须执行此步骤的人员。
- 签出完成后，你将获得其他链接来设置产品和服务。

## <a name="proposal-terms"></a>建议条款

下表包含建议和建议网站上显示的条款和定义。

| Term | 定义 |
|---|---|
| 计费帐户 | 用于管理帐户设置、发票、付款方式和产品的帐户。 |
| 计费配置文件 | 有关组织的信息，可让你自定义发票上包含的项目，以及发票的付款方式。 计费配置文件包括计费帐户名称、用于特定计费配置文件的付款方式、联系人信息、发票设置以及允许您更改计费配置文件、支付账单以及购买产品和服务的权限。 |
| 现有协议 | 组织已与 Microsoft 签订的任何协议。 这可以包括但不限于企业协议、Microsoft 产品&服务协议或Microsoft 客户协议。 |
| Microsoft 客户协议 (MCA)  | 一个协议，其中概述了组织与 Microsoft 持有的帐户的条款和条件。 |
| Microsoft 代表 | 为你和你的组织准备建议的授权 Microsoft 代表。 |
| 组织 | 使用 Microsoft 产品、技术或服务的法律实体。 |
| 准备者 | 准备提案的 Microsoft 代表的电子邮件地址。 |
| 补充条款 | 对包含特定于组织条款的 MCA 的修订。 若要接受补充条款，必须使用 DocuSign 记录电子签名。 |

## <a name="step-1-review-organization-information"></a>步骤 1：查看组织信息

登录后，首先要查看组织的信息。

### <a name="your-organization"></a>你的组织

“ **组织** ”部分显示与之关联的计费帐户。 计费帐户信息是从现有计费帐户提取的，或者由 Microsoft 代表为你创建。 如果你的组织是另一个组织的附属组织，你还会看到一个 **潜在顾客组织** 部分，其中包含该组织的名称和地址。

如果这是组织首次与 Microsoft 建立商业关系，但尚未签署 MCA，则如果 **组织** 或 **潜在顾客组织** 下的信息不正确，请与代表联系以进行更改。 接受 MCA 后，可以在Microsoft 365 管理中心的“[计费帐户](https://go.microsoft.com/fwlink/p/?linkid=2084771)”页上查看和更改组织的地址和联系信息。 如果组织名称更改，请打开服务请求以更新它。 [了解如何打开服务请求](../admin/get-help-support.md)。

### <a name="your-information"></a>你的信息

如果你是新客户，请在 **信息** 下输入姓名、电子邮件地址和电话号码，然后选择 **“保存**”。 如果你是现有客户，请验证信息是否正确。 若要进行任何更正，请选择 **“编辑**”，进行必要的更改，然后选择 **“保存**”。

准备就绪后，选择 **“继续** ”转到下一步。

## <a name="step-2-review-proposal"></a>步骤 2：查看建议

该建议包含与 Microsoft 代表讨论的项目的详细信息。 可以使用建议链接转发电子邮件，以便与组织中的其他利益干系人共享。 使用该链接的其他人对该建议具有只读视图。

如果要在审阅后对建议进行任何更改，请与 Microsoft 代表联系。

### <a name="proposal-contents"></a>建议内容

该建议包含以下信息：

| 节 | 说明 |
|---|---|
| 组织名称 | 为其准备提案的组织的名称。 |
| 有效期至日期 | 建议套餐到期的日期。 如果错过了此到期日期，请联系你的 Microsoft 代表，让他们知道你仍然对该建议感兴趣。 |
| 货币 | 用于计算提案中项成本的货币。 |
| 已准备好 | 请求建议的用户的计费帐户名称、地址、联系人电子邮件地址和电话号码。 |
| 准备者 | 准备提案的 Microsoft 代表的电子邮件地址。 |
| 摘要 | 显示与建议关联的分类汇总。 如有必要，还将显示用于计算成本的外汇 () 汇率。 |
| 建议行项 | 本部分包含提案中包含的所有项目的数量、单价和分类汇总。 |
| 后续步骤 | 本部分指示必须执行的必要操作。 |

若要对 MCA 进行签名，请选择 **“下一步**”下的按钮。 如果必须对补充条款进行签名，则链接会将你带到 DocuSign 站点，在该站点中，你将按照步骤对文档进行签名。

签署任何必要的协议或补充条款后，选择 **“转到签出**”。

## <a name="step-3-checkout"></a>步骤 3：签出

结帐页包含以下部分：

### <a name="sold-to"></a>售达地址

本部分显示用于提案的计费帐户。 如果需要更改任何信息，请选择 **“编辑** ”链接。 还可以使用 **“编辑** ”链接添加组织的税务 ID。 税务 ID 必须与“ **待售到** ”部分中列出的国家/地区相关。 如果你有免税，则必须开具支持票证才能申请免税状态。

若要详细了解税务 ID 以及如何申请免税状态，请参阅 [税务信息](billing-and-payments/tax-information.md)。

### <a name="billed-to"></a>计费到

本部分显示用于确定发票中包含哪些项的计费配置文件，以及如何支付发票。 每个计费周期都会收到每个计费配置文件的单独发票。 可以使用支票或电汇或 Azure 预付款来支付发票费用。 如果还没有计费配置文件，Microsoft 代表会为你创建一个。 签出期间，可以选择其他计费配置文件（如果有）、更改计费配置文件的名称或添加 P.O。 数量。 还可以创建新的计费配置文件。

有关计费配置文件的信息，请参阅 [“管理计费配置文件](billing-and-payments/manage-billing-profiles.md)”。

### <a name="proposal-items-in-this-order"></a>按此顺序排列的建议项

本部分显示建议中包含的所有项的列表。 该列表可以包括以下一个或多个类别：

- **补充条款** 包含组织条款的任何 MCA 修订的列表。 例如，此列表可能包含 HIPAA 或 GDPR 术语。
- **立即购买** 在建议接受工作流结束时结帐时支付的项列表。
- **适用于未来费用的折扣 ()** 作为建议的一部分收到的折扣列表。
- **包括** 作为建议包的一部分包含的项目列表，不收取额外费用。 其中一些项将来可能会产生与之关联的成本。

### <a name="summary"></a>摘要

本部分显示要支付的项目数、小计、估计税款和订单总金额。

若要放置订单，请选择 **“放置”订单** 或 **“接受协议 &amp; ”位置顺序**。

下订单后，将收到确认，并需执行后续步骤。 如果购买了 Azure 计划，下一步是在Azure 门户中设置计费帐户。

## <a name="step-4-set-up-your-new-billing-account-azure-customers-only"></a>步骤 4：仅)  (Azure 客户设置新的计费帐户

如果你是新客户，并且已购买 Azure 产品作为建议的一部分，下一步是设置新的计费帐户。 若要了解如何操作，请参阅[为Microsoft 客户协议设置计费帐户](/azure/cost-management-billing/manage/mca-setup-account)。

如果你是具有企业协议的现有 Azure 客户，并且是首次签署 MCA，下一步是了解协议之间的更改，以及如何使用新的计费帐户完成任务。 若要了解详细信息，请参阅[Microsoft 客户协议的计费帐户中完成企业协议任务](/azure/cost-management-billing/manage/mca-enterprise-operations)。

## <a name="understand-invoicing"></a>了解 Invoicing

签出并完成订单后，会在 24-48 小时内发送初始发票。 之后，你将每月 5 日左右收到发票。 每月发票包含上个月的费用。 如果你的帐户有任何额度，则会从计费对象的货币额度中扣除这些额度，并将其应用于发票余额。 应用额度后的剩余余额是到期余额。 您有 30 天的时间从计费日期开始支付发票。

发票的 PDF 副本中包含用于发送支票或电汇的付款说明。 若要查看或下载发票，请 [参阅查看帐单或发票](billing-and-payments/view-your-bill-or-invoice.md)。
