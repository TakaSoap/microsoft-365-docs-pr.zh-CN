# <a name="deploy-microsoft-365-lighthouse-baselines"></a>部署Microsoft 365 Lighthouse基线

Microsoft 365 Lighthouse部署标准托管租户配置，以确保客户租户中的用户、设备和数据安全。 有七 [种默认基线配置](m365-lighthouse-deploy-standard-tenant-configurations-overview.md) 是 Lighthouse 的标准配置。 使用 Lighthouse 部署计划功能，可以查看、测试和部署所有租户的安全配置。 部署计划仅适用于活动租户。 载入租户后，可以将客户的当前配置与默认基线配置进行比较，并采取适当的措施。

## <a name="before-you-begin"></a>准备工作

确保你和客户租户满足要求 [for Microsoft 365 Lighthouse](m365-lighthouse-requirements.md)。

## <a name="view-a-deployment-plan"></a>查看部署计划

1. 在左侧导航页中，选择 **"租户"**。

2. 从租户列表中选择要查看的租户。

3. 选择" **部署计划"** 选项卡。

    "部署计划"选项卡提供了租户的部署计划中包含的每个部署步骤的可搜索和可导出列表，其中包括每个部署步骤的以下信息：

    | 列            | 说明 |
    |-----------------|-------------------------------------------------------------------------------------|
    | 部署步骤 | 部署步骤的说明。                                                     |
    | 状态          | 部署步骤的状态。                                                  |
    | 基线        | 从中派生部署步骤的基线。                             |
    | 类别        | 部署步骤是否与管理设备、标识或数据相关联。 |
    | 上次更新    | 上次更新部署步骤的日期。                             |

4. 从列表中选择要查看的部署步骤。

    "部署步骤"页提供以下信息：

    | 列            | 说明 |
    |-------------------|-----------------------------------------------------------------------------------------------|
    | 摘要        | 部署步骤用途的摘要。                                         |
    | 基线       | 从中派生部署步骤的基线。                             |
    | 类别       | 部署步骤是否与管理设备、标识或数据相关联。 |
    | 所需的 SKU   | 完成部署步骤所需的 SKUS。                                      |
    | 用户影响    | 将步骤部署到租户用户的影响。                             |
    | 对于用户 | 指向租户用户可能发现有用的资源的链接。                             |
    | 后续步骤     | 有关任何适用的下一步步骤的链接和指导。                                |

    部署步骤由一个或多个流程组成，需要完成这些流程以满足部署步骤的要求。 "部署步骤"页包括列出部署步骤中包含的每个进程的进程表，并提供以下信息：

    | 列            | 说明 |
    |-------------------|-------------------------------------------------------------|
    | 进程名称      | 进程的名称，如果选中，将打开适用的"进程"选项卡。          |
    | 状态            | 检测到部署过程中包含的这些设置配置的状态。           |
    | 管理门户 | 管理与进程关联的配置设置的门户。 |

## <a name="deploy-a-deployment-step"></a>部署部署步骤

1. 在左侧导航页中，选择 **"租户"**。

2. 从租户列表中选择要查看的租户。

3. 选择" **部署计划"** 选项卡。

4. 从"部署步骤"列表中，选择要部署的部署步骤。

5. 选择 **"审阅并部署"**。

6. 从" **确认配置"窗格中** ，选择"部署 **"**。

## <a name="test-a-deployment-step"></a>测试部署步骤

对于通过条件访问策略部署的部署步骤，可以将部署步骤中的配置设置与任何现有策略中的设置进行比较，而无需将设置部署到租户。

1. 在左侧导航页中，选择 **"租户"**。

2. 从租户列表中选择要查看的租户。

3. 选择" **部署计划"** 选项卡。

4. 从"部署步骤"列表中，选择要部署的部署步骤。

5. 选择 **"审阅并部署"**。

6. 从" **确认配置"** 窗格中，选择 **"在不部署的情况下测试这些设置"**。

7. 选择 **"测试"**。

"确认配置"窗格将关闭并显示策略比较。 现有租户内的每个策略都将在"检测到的设置"表中列出。

"检测到的设置"表列出了每个现有策略，并汇总了设置数以及以下状态之一的用户数（括号）：

| 状态         | 说明
|-------------|------------------------------------------------------------|
| 等设置       | 部署计划中具有等效值的租户中的配置设置的总数。      |
| 缺少设置     | 部署计划中缺少租户中的值的配置设置的总数。      |
| 冲突设置 | 部署计划中在租户中具有冲突值的配置设置的总数。 |

还可以在模块化表中查看检测到的设置，该表提供设置和用户级别每个策略的配置设置详细信息，并可以按以下每个设置状态进行排序：

| 状态         | 说明
|-------------|------------------------------------------------------------|
| 设置总数       | 部署过程中包括的配置设置总数。                        |
| 等设置       | 部署计划中具有等效值的租户中的配置设置的总数。      |
| 缺少设置     | 部署计划中缺少租户中的值的配置设置的总数。      |
| 冲突设置 | 部署计划中在租户中具有冲突值的配置设置的总数。 |
| 额外设置       | 在租户中具有值但在部署计划中没有值的配置设置的总数。     |

进行此比较后，Lighthouse 将自动更新"检测到的状态、部署状态和部署步骤"状态。

如果没有要比较的现有策略，请选择"查看并部署"以重新打开"确认配置"窗格，然后选择"部署"。

如果存在要与之比较的现有策略，您可以：

- 编辑部署计划的配置设置，然后根据现有策略重新测试这些设置，选择"复查并部署"以重新打开"确认配置"窗格，调整所需的配置设置，重新选中复选框，然后选择窗格底部的"测试"。

- 在适用的管理门户中编辑现有策略，以通过以下两者之一协调差异：
  - 应用缺少的设置
  - 编辑冲突设置
  - 删除现有策略

对于可通过"灯楼"自动执行的每个部署过程，都同时存在部署状态和检测到的状态。

- 检测到的状态指示当前部署此进程中的设置的程度。
- 部署状态是上次部署到租户的状态。

无论现有策略如何，都可以部署部署步骤，但在没有冲突设置之前不会被视为完成部署步骤。 未能解决这些冲突设置可能会影响用户体验。 

如果现有策略中的租户存在相同的设置，则部署部署步骤的实例将导致租户中的现有设置重复，但不会影响用户体验。 

提供了额外设置，让你了解，但不要求你采取措施。

有关策略冲突管理详细信息，请参阅条件[Azure AD访问文档](/azure/active-directory/conditional-access/)。

## <a name="update-deployment-step-status"></a>更新部署步骤状态

1. 在左侧导航页中，选择 **"租户"**。

2. 从租户列表中选择要查看的租户。

3. 选择" **部署计划"** 选项卡。

4. 从部署步骤列表中，选择要更新的部署步骤。

5. 从 **"地址** "下拉列表中，选择操作状态。

    | 操作状态                        | 说明      |
    |---------------------------------------|----------------------------------------|
    | To address                        | 不包含多个部署步骤过程的所有部署步骤的默认状态。      |
    | Planned                           | 部署步骤已经过规划，但尚未完成。                                      |
    | 接受的风险                     | 用户已接受通过应用部署步骤本该避免的风险。 |
    | 通过第三方解决的风险 | 通过实施第三方应用程序或软件解决了风险。             |
    | 通过备用方法解决  | 通过替代方法（如实现内部工具）解决了风险。    |
    | 手动配置已应用      | 已手动应用部署计划中指定的配置。                         |

## <a name="share-deployment-step"></a>共享部署步骤

1. 在左侧导航页中，选择 **"租户"**。

2. 从租户列表中选择要查看的租户。

3. 选择" **部署计划"** 选项卡。

4. 从"部署步骤"列表中，选择要共享的部署步骤。

5. 从 **"共享** "下拉列表中，选择以下选项之一。

    | 选项  | 说明 |
    |-----------|-------------------------------------------------------------------------|
    | 复制  | 将指向部署步骤的链接复制到剪贴板。                                     |
    | 电子邮件 | 在本地计算机上打开新电子邮件，并插入部署步骤的链接。 |

    此链接将允许组织中具有权限的任何人查看租户的部署计划。
